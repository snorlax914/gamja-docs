# gamja-docs 버그 점검 보고서

작성일: 2026-05-16

## 1. 발생한 에러

`GET /documents` 호출 시 500 에러.

```
pydantic_core._pydantic_core.ValidationError: 1 validation error for DocumentInfo
classification.reason
  Field required [type=missing, input_value={'category': '편지/공...', 'category_code': 'letter', 'confidence': 0.85}, input_type=dict]
```

### 원인
- `Classification` 스키마(`backend/app/schemas.py:12-17`)는 `reason: str` 필수.
- 그러나 LLM(qwen3:4b)이 분류 JSON을 만들 때 `reason` 키 자체를 출력하지 않은 케이스 발생.
- `OllamaService.classify()`(`backend/app/services/llm.py:62-84`)는 JSON 파싱 성공 시 dict를 그대로 반환 → 누락된 채 `_docs.json`에 저장됨.
- 다음 `GET /documents`에서 `DocumentInfo(**d)` 변환 시 검증 실패.

### `reason`이 비어 있던 이유 (추정)
1. **qwen3:4b 모델 한계**: 작은 모델은 JSON 스키마 키를 가끔 누락. thinking 단계에서 "분류했다"고 판단하면 부가 필드(reason 같은 거)를 마지막 출력에서 빼먹는 경향이 있음.
2. **프롬프트 모호**: `CLASSIFY_PROMPT`(`llm.py:11-32`)가 `<한 줄 이유>`라고만 적혀있고 "필수"라는 표현이 없음.

## 2. 적용한 수정

`backend/app/schemas.py:16` — `reason`을 Optional로 완화.

```python
class Classification(BaseModel):
    category: str
    category_code: str
    confidence: float
    reason: Optional[str] = ""   # ← was: reason: str
```

이로써 기존 저장된 문서도 살아나고, 향후 LLM이 또 빼먹어도 500이 아닌 정상 응답이 가능.

## 3. 유사 버그 발생 가능성 (미수정, 후속 작업 권장)

### High — 거의 확실히 재발

**#1. `documents.py:147-152` — `filename=None` 저장 가능**
```python
store.add(doc_id, filename=file.filename, ...)
```
`UploadFile.filename`은 `Optional[str]`. None이면 `_docs.json`에 그대로 들어가고, `DocumentInfo.filename: str` 필수라 list 호출이 또 500. 응답 객체엔 `file.filename or ""`(line 158) 처리가 돼있지만 store엔 누락.
권장: `filename=file.filename or "unnamed"` 로 수정.

**#2. `classify()` 다른 키도 누락 가능**
이번엔 `reason`이었지만 다음엔 `category` / `category_code` / `confidence`가 빠질 수도 있음. classify의 fallback(`llm.py:78-84`)은 **JSON 파싱 실패시에만** 동작.
권장: `classify()` 끝에 `.get()`으로 모든 필수 키 기본값 보강.
```python
return {
    "category": parsed.get("category", "기타"),
    "category_code": parsed.get("category_code", "other"),
    "confidence": parsed.get("confidence", 0.0),
    "reason": parsed.get("reason", ""),
}
```

### Mid

**#3. `vector_store.py:79` — `r.payload["text"]` KeyError 가능**
`chunk_idx`만 `.get()` 쓰고 `text`는 `[]`. 외부 손상/스키마 변경 시 KeyError → 500.
권장: `r.payload.get("text", "")`로 통일.

**#4. `_docs.json` 스키마 진화에 취약**
`DocStore._load()`(`doc_store.py:18-20`)는 검증 없이 dict 로드. `DocumentInfo`에 필수 필드 추가 시 **기존 저장된 문서들 전부** 다음 list 호출에서 깨짐.
권장: 새 필드는 항상 `Optional`로 시작 / 또는 list_documents에서 검증 실패 문서만 스킵하고 로그.

### Low

**#5. `Classification`의 `category`/`category_code`/`confidence`도 여전히 필수**
이번에 `reason`만 Optional로 풀었음. #2 fix를 안 하면 이 키들에서 다시 동일 패턴 에러.

## 4. 별개로 식별된 이슈 (참고)

### 첫 질문 응답이 늦거나 안 오는 현상
- 원인: Ollama qwen3:4b 콜드 스타트 + (선택적으로) thinking 단계 지연.
- 두 번째 질문부터 빨라지는 것으로 콜드 스타트 확정.
- 권장 수정 (미적용):
  - `llm.py`의 Ollama 호출 옵션에 `"keep_alive": -1` 추가 → 모델을 VRAM에 상시 유지 (~2.5GB 점유)
  - `main.py`의 lifespan에 워밍업 호출 추가 → 서버 시작 직후도 hot
